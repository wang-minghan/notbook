# 岗位分析模板

---

## JD解读模板

拿到JD后，按以下结构拆解：

```
1. 岗位核心定位
   这个岗位存在的目的是什么？解决什么业务问题？
   → 从JD的第一段和岗位名称推断

2. 硬性门槛 vs 软性加分
   硬性门槛："必须有"、"X年以上经验"、"XX学历"
   软性加分："优先考虑"、"有XX经验者优先"、"熟悉XX者加分"
   → 硬性门槛不达标通常不建议投；软性加分达不到不影响

3. 核心能力要求（按出现频次排序）
   JD中反复出现的词 = 真正的核心要求
   只出现一次的词 = 可能是HR凑字数的

4. 隐含信息
   JD写得详细 → 岗位成熟，看经验匹配度
   JD写得模糊 → 岗位新设，看潜力和态度
   强调"抗压"/"快节奏" → 工作强度大
   强调"自驱"/"独立" → 管理松散，靠自己
   薪资范围跨度大 → 对候选人层次有弹性

5. 公司背景快速调研
   公司规模、融资阶段、核心业务
   最近的新闻和动态
   在招岗位的数量（大量招人=快速扩张或人员流动大）
```

## 匹配度评估模板

```
JD核心要求         我的匹配情况        差距/应对策略
─────────────────────────────────────────────────────
要求1：[X]        □ 完全匹配          
                  □ 部分匹配，差距在[Y]  → 面试中如何弥补
                  □ 不匹配            → 是否是硬伤

要求2：[X]        □ 完全匹配
                  □ ...

要求3：[X]        □ ...

综合判断：
  □ 强匹配（3个以上核心要求完全匹配）→ 重点投
  □ 有差距但可争取（1-2个核心差距可弥补）→ 有针对性地准备
  □ 硬伤较多（核心要求不匹配）→ 建议换方向
```

## 行业关键词速查

### 互联网/科技

```
产品：需求分析、用户画像、A/B测试、MVP、PMF、DAU/MAU、转化率、
      留存率、漏斗分析、用户体验、竞品分析、PRD、版本迭代
技术：微服务、分布式、高并发、容器化、CI/CD、DevOps、
      云原生、中间件、数据库优化、性能调优
运营：增长黑客、AARRR、用户生命周期、拉新/促活/留存、
      内容矩阵、私域流量、裂变、ROI、LTV、CAC
```

### 金融

```
投行：IPO、并购、尽职调查、财务建模、估值、DCF、
      投资备忘录、交割、资本市场
基金：投资研究、行业分析、项目筛选、投后管理、
      收益率、风险控制、资产配置
银行：信贷审批、风控模型、合规、KYC、AML、
      零售银行、对公业务、资产质量
```

### 快消/零售

```
品牌管理、渠道策略、消费者洞察、新品开发、
动销率、铺货率、终端管理、促销ROI、
供应链管理、库存周转、品类管理
```

### 咨询

```
商业分析、市场规模估算、竞争格局、价值链分析、
客户访谈、假设驱动、MECE、金字塔原则、
项目管理、交付物、工作流
```
