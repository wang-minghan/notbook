# knowledge-base-lifecycle

这页记录知识库从 raw 到 wiki 的编译与维护循环。

## 生命周期
- raw：原始资料，尽量只读
- ingest：把资料编译成 entity / concept / comparison / query
- wiki：持续维护的编译产物
- query：高质量问答结果回写知识库
- workflow：可重复执行的行动模板
- lint：检查孤岛、断链、冲突、缺页
- refine：先最小修复，再做重构与合并

## 维护目标
- 每个关键页至少 2 个相关链接
- 重要枢纽页必须双向链接
- 重要结论不要只停留在聊天中
- 优先效果最优，不追求最小改动
