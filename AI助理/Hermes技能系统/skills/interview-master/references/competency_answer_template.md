# 能力域拆解与深度答案准备指南

## 一、能力域拆解方法论

### 拆解逻辑
1. 从JD提取考察关键词 → 归类为能力域
2. 结合公司调研判断每个能力域的权重（该公司当前最需要什么）
3. 对照用户背景匹配经历
4. 识别高概率题目

### 常见岗位的能力域参考（Claude必须根据具体JD动态调整，不能直接套用）

## 二、能力-经历匹配矩阵模板

```
╔══════════════════════════════════════════════════════════╗
║  能力-经历匹配矩阵                                       ║
╠══════════════╦═══════════════╦═══════════╦═══════════════╣
║  能力域       ║ 你的匹配经历   ║ 匹配强度   ║ 准备建议      ║
╠══════════════╬═══════════════╬═══════════╬═══════════════╣
║ [能力域1]    ║ [具体项目/     ║ ★★★★★   ║ 核心弹药，打磨 ║
║              ║  经历]         ║           ║ 数据细节       ║
╠══════════════╬═══════════════╬═══════════╬═══════════════╣
║ [能力域2]    ║ [具体项目/     ║ ★★★★☆   ║ [具体建议]     ║
║              ║  经历]         ║           ║               ║
╠══════════════╬═══════════════╬═══════════╬═══════════════╣
║ [能力域3]    ║ [经历偏少]     ║ ★★★☆☆   ║ ⚠️ 风险项      ║
║              ║               ║           ║ [补充策略]     ║
╚══════════════╩═══════════════╩═══════════╩═══════════════╝

⚠️ 准备盲区提醒：
- [能力域X]你缺少直接经验，建议用[替代策略]来应对
- [能力域Y]你有经验但缺具体数据，面试前务必回忆/整理
```

### 匹配强度评分标准
- ★★★★★ — 有直接主导经验 + 可量化结果 + 可复用方法论
- ★★★★☆ — 有相关经验但不够深/不够直接，需要补充细节
- ★★★☆☆ — 有间接经验或理论理解，需要转化表达方式
- ★★☆☆☆ — 经验很弱，需要准备替代案例或坦诚 + 学习意愿
- ★☆☆☆☆ — 完全空白，面试中应主动引导到强项

### 弱项应对策略
当某个能力域匹配度≤★★★☆☆时，Claude必须给出以下之一：
1. **迁移策略**：从其他经历中找到可迁移的案例，换个角度讲述
2. **方法论补位**：虽然没做过，但对方法论有深入理解（适合AB测试等方法论重的领域）
3. **坦诚+学习路径**：承认经验不足 + 说清楚你的学习计划和已有基础
4. **引导转移**：在回答中自然过渡到你强的能力域

## 四、答案质量检查清单

每道答案必须通过以下检查：

- [ ] 有没有至少2个具体数据点？（"提升了转化率"不算，"从3%提升到5.8%"才算）
- [ ] 有没有体现你的决策过程？（不只是做了什么，更是为什么这么做）
- [ ] 有没有时间控制标注？（每段几秒，总共不超过3分钟）
- [ ] 有没有"只有你能说出来"的细节？（通用化的答案 = 弱答案）
- [ ] 有没有和公司调研做连接？（体现你对这家公司的理解）
- [ ] 追问预判覆盖了没有？（至少3个方向）
- [ ] 有没有标注用户需要自己补充的信息？（Claude不能编造用户的数据）
- [ ] 开场是否在15秒内定位清楚？（防止"从头说起"的流水账）
- [ ] 结尾是否有升华/方法论？（不能戛然而止）

## 五、答案框架选择指南

不同类型的题目使用不同框架：

| 题目类型 | 推荐框架 | 核心要求 |
|---------|---------|---------|
| 行为问题（过去经历） | STAR（情境-任务-行动-结果） | 结果必须量化 |
| 动机问题（为什么来） | 3W（Why role / Why company / Why now） | 必须结合公司调研 |
| 思路问题（怎么解决） | 金字塔（结论先行 + 3条依据） | 先说答案再说过程 |
| 优劣势问题 | CAR（Context + Action + Result/Learning） | 缺点要真实但不致命 |
| 数据分析题 | MECE拆解 + 假设验证 | 先问清口径再分析 |
| 情景模拟题 | 利益相关者分析 + 决策树 | 考虑多方立场 |
| 商业判断题 | 第一性原理 + 数据支撑 | 有立场、敢判断 |
