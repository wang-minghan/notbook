# 北京灵易数智科技有限公司 — 算法工程师

**时间：** 2025.04 — 2025.08  
**职位：** 算法工程师  

---

## 核心职责

围绕复杂工程数据与多源异构数据建模问题，系统开展数据处理、模型建模、降维与推理优化等核心算法工作，覆盖从数据治理到模型部署的完整技术链路。

## 主要工作

### 1. 数据处理与特征工程
- 设计并落地端到端的数据处理流程
- 覆盖：数据采集 → 清洗 → 规范化 → 特征构造 → 训练集构建
- 提升数据一致性与模型训练稳定性

### 2. 高维小样本建模
- 研究并实现多种降阶与代理建模方法
- 通过算法选择与特征压缩，在保证精度的前提下降低复杂度与计算成本

### 3. 模型服务化
- 设计并实现模型服务化 API 接口
- 支持前后端解耦与异步任务调度
- 提升模型推理吞吐能力与系统整体响应性能

### 4. POD 降维建模
- 实现基于 POD（Proper Orthogonal Decomposition）的降维方案
- 通过能量阈值控制有效模态数量
- 降低训练复杂度的同时显著提升模型泛化能力与数值稳定性

### 5. 多类型数据统一处理
- 面向场数据与离散数据的分布特征
- 构建统一且可扩展的数据加载与处理机制
- 结合数据清洗、异常值处理与归一化策略

### 6. 多方法融合建模
- 综合运用传统代理模型与深度学习方法：
  - RBF（径向基函数）
  - Kriging（克里金）
  - DNN（深度神经网络）
  - **KAN**（Kolmogorov–Arnold Network）
- 根据不同数据规模与非线性特征选择最优策略

### 7. 输出可解释性
- 通过反归一化与物理量约束处理
- 确保模型输出在工程场景下的可解释性与可落地性

## 主要业绩

1. 核心模型在多类工程数据集上**预测精度稳定达到 95% 以上**
2. 构建的模型服务与系统架构支持**水平扩展与模块化集成**
3. 灵活适配多种 CAE 软件体系与不同业务场景

## 技术栈

`Python` `POD` `RBF` `Kriging` `DNN` `KAN` `Docker` `RESTful API`
