## 结构约定（给维护者）

- **目录负责粗分类，链接负责语义关系**；不靠文件夹层级假装知识层级。
- **页面类型**：entity / concept / comparison / query / overview / workflow / time / generated。聪明模型无需背清单，按语义自然归类即可。
- **scope 三层**：`global` 长期稳定会被反复引用；`section` 仅在某子目录成立；`local` 时间页 / 临时页。
- **索引纪律**：`index.md` 只收 `global` 页；日计划 / 周计划等 time 页只进各自 section 索引，不进全局。
- **模块索引命名**：`index-<模块名>.md`，仅当该模块有 ≥2 个真实子页 / 子模块时才存在。单子页模块不建索引桩。
- **增量维护**：每次结构性改动回写 `log.md`，只写“变了什么 / 为什么”，不重写全库总结。
- 出现冲突保留并标记 `contested`，不覆盖。
