# 简历逐项检查清单

---

## 整体诊断（30秒扫描）

| 检查项 | 标准 | 常见问题 |
|--------|------|---------|
| 排版 | 干净整齐、模块清晰、1-2页 | 花哨模板、3页以上、密密麻麻 |
| 岗位序列 | 职业路径清晰可读 | 频繁跳槽无逻辑、岗位跨度大无解释 |
| 关键词匹配 | 与目标JD核心词对应 | 通用简历投所有岗位 |
| 第一印象 | 30秒内能看到核心价值 | 重要信息埋在最后 |

---

## 逐项检查

### 个人信息区

```
□ 姓名、电话、邮箱（必须）
□ 求职意向/目标岗位（强烈建议写）
□ 城市/是否接受异地（社招重要）
□ LinkedIn/GitHub/作品集链接（加分项）
□ 不要写：身份证号、民族、婚育状况（除非JD明确要求）
□ 照片：外企建议放，互联网公司可放可不放，证件照即可
```

### 工作/实习经历区（最重要）

**每段经历的检查点：**

```
□ 公司名称 + 岗位名称 + 起止时间 → 是否清晰？
□ 一句话说明这段经历的核心价值 → 能否10秒内看懂你做了什么？
□ 每条bullet point是否包含"动词+行动+结果"？
□ 结果是否可量化？（数字、百分比、金额、排名）
□ 是否区分了"主导"和"参与"？
□ 强动词是否到位？
```

**强动词替换表：**

| 弱动词 | 替换为 |
|--------|--------|
| 负责 | 主导、搭建、设计、推动 |
| 参与 | 协助主导、独立完成[某模块] |
| 做了 | 策划并执行、设计并上线 |
| 配合 | 联动[X]团队完成 |
| 优化 | 重构[X]流程，[指标]提升[Y]% |

**量化改写示例：**

```
❌ 负责用户增长相关工作，效果显著
✅ 主导用户增长策略，通过裂变活动+渠道投放组合拳，
   6个月内新增注册用户50万，获客成本从35元降至12元

❌ 参与了公司核心产品的迭代优化
✅ 独立负责支付流程重构，减少3步冗余操作，
   支付成功率从82%提升至94%，月均挽回GMV约200万

❌ 完成了多个技术项目的开发
✅ 主导搜索服务性能优化，引入倒排索引+缓存预热方案，
   P99响应时间从800ms降至120ms，QPS提升6倍
```

### 教育背景区

```
□ 学校 + 专业 + 学位 + 毕业时间
□ GPA如果不错（3.5+/4.0或专业前20%）可以写
□ 相关课程：只写和目标岗位相关的核心课程
□ 学术成果：论文、专利（技术岗加分）
```

### 技能/证书区

```
□ 只列和岗位相关的技能
□ 区分熟练度：精通 / 熟练 / 了解（不要全写精通）
□ 证书只列有认知度的（CPA/PMP/AWS认证等）
□ 不要写：Office全家桶（除非是行政岗）
```

### 项目经历区（可选，适合经历不够丰富时补充）

```
□ 项目名称 + 你的角色 + 时间
□ 项目背景一句话
□ 你的核心贡献和结果
□ 和正式工作经历不重复
```

## 常见致命问题速查

| 问题 | 严重程度 | 解决方案 |
|------|---------|---------|
| 没有可量化的结果 | ⚠️⚠️⚠️ | 每段经历至少一个数字 |
| 通用简历投所有岗位 | ⚠️⚠️⚠️ | 每个岗位定制关键词 |
| 只写"做了什么"没写"做出了什么" | ⚠️⚠️⚠️ | 加结果和业务影响 |
| 时间线有空白未解释 | ⚠️⚠️ | 简要说明（进修/家庭等） |
| 篇幅过长（3页以上） | ⚠️⚠️ | 砍掉5年前的详细描述 |
| 排版混乱/字体不统一 | ⚠️ | 用标准模板重排 |
| 错别字/语法错误 | ⚠️ | 逐字检查或让人帮审 |

## ATS（简历自动筛选系统）适配要点

很多公司使用ATS系统自动筛选简历，简历格式不对可能直接被过滤掉：

```
□ 用标准的.pdf或.docx格式（不要用图片/PPT格式的简历）
□ 不要用表格/多列布局（ATS可能解析错乱）
□ 关键技能词用JD中的原词（ATS按关键词匹配）
□ 标题用标准名称（"工作经历"而不是"我的故事"）
□ 不要把关键信息放在页眉/页脚（部分ATS读不到）
```
