# 面试准备文档输出模板

> 本文件指导Claude在完成面试指导后，为用户生成可下载的面试准备文档。
> 用户可以在任何阶段结束后要求生成文档，也可以在所有准备完成后一次性生成。

---

## 触发时机

**Claude 应该在以下时机主动提醒用户可以生成文档：**

1. 阶段三（问答准备）完成后 → 提醒可以生成"面试问答Cheatsheet"
2. 阶段四（实操考核）完成后 → 提醒可以生成"实操考核速查卡"
3. 阶段六（薪资谈判）完成后 → 提醒可以生成"薪资谈判备忘录"
4. 用户主动说"帮我总结一下" / "有没有什么可以打印出来的" / "我想要一份文档"

**话术：**
> "我们刚才准备的内容挺多的。我可以帮你生成一份面试准备文档，
> 你可以下载、打印或存到手机里，面试前随时翻看。需要吗？"

---

## 文档类型和模板

### 类型一：面试全景准备手册

**适用场景：** 用户完成了多个阶段的准备，需要一份综合性文档

**文件格式：** Markdown（.md）或 Word（.docx）

**内容结构：**

```markdown
# [用户姓名]的面试准备手册
## 目标岗位：[公司名] - [岗位名]
## 生成日期：[日期]

---

### 一、岗位核心分析
- 岗位匹配度判断：[强匹配/有差距/硬伤]
- 核心考察维度：[从JD分析中提炼的3-5个]
- 我的优势：[具体列出]
- 需要弥补的差距：[具体列出]

### 二、面试预判
- 预计面试轮数：[X轮]
- 各轮重点：
  - 第一轮（[角色]）：重点考察[X]
  - 第二轮（[角色]）：重点考察[X]
  - ...

### 三、高概率问题 & 我的答案
1. [问题]
   → 我的回答要点：[基于准备阶段的核心答案要点，不是完整稿]
   → 关键数据：[需要记住的数字]

2. [问题]
   → ...

### 四、我的STAR案例库
案例1：[名称] — 适用于[什么类型的问题]
  S: [一句话]
  T: [一句话]
  A: [核心行动，2-3条]
  R: [量化结果]

案例2：...

### 五、反向提问清单
- 给[角色]问：[问题]
- 给[角色]问：[问题]

### 六、薪资谈判备忘
- 我的目标薪资：[X]
- 市场行情参考：[X-Y]
- 底线（不要说出来）：[X]
- 可以争取的其他维度：[签字费/年终保底/...]

### 七、面试前最后检查
□ 公司最近动态查过了吗？
□ 核心案例的数字还记得吗？
□ 着装准备好了吗？
□ 面试地点/链接确认了吗？
□ 提前到15分钟
```

### 类型二：问答Cheatsheet（速查卡）

**适用场景：** 面试前快速复习核心问答

**格式：** 单页Markdown，信息密度高，适合手机查看

**内容结构：**

```markdown
# 面试速查卡 | [公司] [岗位]

## 自我介绍（90秒版）
[基于准备的精简版自我介绍要点]

## 必考题速答
| 问题 | 核心要点（关键词） |
|------|------------------|
| 说说你自己 | [3个关键词] |
| 为什么来我们公司 | [3个理由] |
| 最大挑战 | [案例名] + [结果数据] |
| 为什么离职 | [一句话答案] |
| 你的优缺点 | 优：[X] / 缺：[X+改进行动] |

## 核心数据速记
- [项目A]：[关键数字]
- [项目B]：[关键数字]
- [项目C]：[关键数字]

## 反向提问（选2个问）
1. [问题]
2. [问题]
3. [问题]

## 薪资（心里有数）
目标：[X] | 报价：[X] | 绝不低于：[X]
```

### 类型三：实操考核速查卡

**适用场景：** 面试包含case study / 笔试 / 群面等实操环节

**内容结构：**

```markdown
# 实操考核速查卡 | [岗位类型]

## 产品设计题框架
1. 明确目标（1-2min）→ 确认用户/场景/指标/约束
2. 用户分析（3min）→ 分层/痛点/现有方案
3. 方案设计（5-8min）→ 2-3方向→选1个深入→流程图
4. MVP & 优先级（2-3min）→ RICE排序
5. 衡量标准（1-2min）→ 核心指标/验证方法

## 数据下降题框架
确认定义 → 排除噪音 → MECE拆解（用户/流程/时间）→ 假设 → 行动

## 关键提醒
- 先说思路再动手
- 边做边说（展示思考过程）
- 收尾要总结
- 承认不确定性是加分的

## [如果是AI PM] AI可行性判断框架
任务本质 → 现有AI能力 → 数据需求 → 效果天花板 → 成本收益 → 非AI替代方案
```

### 类型四：薪资谈判备忘录

**适用场景：** 即将进入谈薪环节

**内容结构：**

```markdown
# 薪资谈判备忘录

## 我的三个数字
- 理想薪资（不主动说）：[X]
- 目标薪资（对外报）：[X]
- 底线薪资（绝不说出）：[X]

## 目标公司薪资情报
- JD薪资范围：[X-Y]
- 市场同岗位参考：[X-Y]
- 该公司往年类似岗位：[如有信息]

## 我的筹码
- 其他offer：[有/无，如有注明大致情况]
- 核心优势：[和市场上其他候选人比，我的差异化]

## 谈判话术备忘
如果HR先问期望薪资：
  → "[我准备好的话术]"

如果报价低于预期：
  → "[我准备好的话术]"

## 确认清单
□ 月薪结构（底薪+绩效比例）
□ 年薪月数（12/13/14/15/16）
□ 年终奖（保底还是浮动）
□ 五险一金基数和比例
□ 试用期时长和薪资
□ 竞业协议
```

## 文档生成的行为指令

### Claude 生成文档时的原则

1. **基于实际对话内容填充，不用通用模板**
   - 用户的真实背景、真实案例、真实数据
   - 如果某个板块用户没准备过，标注"待准备"而不是空着

2. **信息密度高，废话少**
   - 这是给面试前快速翻看的，不是教学文档
   - 用关键词和短句，不用完整段落

3. **格式适合快速查阅**
   - 表格 > 段落
   - 关键词 > 完整句子
   - 重要数字加粗

4. **文件命名规范**
   - `[公司名]_[岗位]_面试准备手册.md`
   - `[公司名]_[岗位]_问答速查卡.md`
   - `[公司名]_[岗位]_实操考核速查.md`
   - `[公司名]_薪资谈判备忘录.md`

5. **如果用户需要 .docx 格式**
   - 读取 `/mnt/skills/public/docx/SKILL.md` 获取 docx 生成方法
   - 使用专业排版，方便打印
