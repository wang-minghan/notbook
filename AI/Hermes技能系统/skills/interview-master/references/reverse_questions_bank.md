# 反向提问题库

> 按面试官角色和轮次分类。每轮面试结束时的提问质量直接影响面试官对你"段位"的判断。

---

## 提问原则

1. **每轮准备2-3个问题，实际问1-2个**（时间不够时至少问1个）
2. **问题要和面试官的角色匹配**（不要向HR问技术架构，不要向技术官问年假）
3. **问题要展示你的思考**（好问题本身就是能力的体现）
4. **不要问一搜就能找到答案的问题**（"你们公司有多少人？"）
5. **不要在早期轮次问薪资福利**（等到HR谈薪环节）

---

## 一、HR/初筛轮

**适合问的：**

```
关于流程：
- "接下来的面试流程是怎样的？大概需要多长时间？"
- "这个岗位目前面试到什么阶段了？有多少候选人在进行中？"

关于岗位：
- "这个岗位是新设的还是有人离职后补招？"
  → 新设岗位意味着可能没有现成流程，需要自己摸索
  → 补招意味着有前任的工作交接和预期对照
- "这个岗位汇报给谁？团队目前有多少人？"

关于公司：
- "公司目前最重视的业务方向是什么？"
- "新人入职后有什么培训或onboarding计划？"
```

**不要问的：**
- ❌ "你们加不加班？""年假多少天？"（显得最关心这个）
- ❌ "你们公司好不好？"（太泛，也让HR难回答）
- ❌ "我表现怎么样？能过吗？"（把HR放在尴尬位置）

---

## 二、业务面试官/直属leader

**适合问的（高分问题）：**

```
关于团队和业务：
- "这个团队目前最大的业务挑战是什么？"
  → 展示你关心业务本身，不只是找份工作
  → 面试官的回答也帮你评估这个岗位

- "如果我入职的话，前3个月您最希望我达成什么？"
  → 展示结果导向的思维
  → 也帮你判断这个岗位的期望是否合理

- "团队目前的技术栈/工作流程是怎样的？有没有想改进的地方？"
  → 展示你想了解实际工作情况
  → 如果你有相关改进经验，可以自然展开

关于个人发展：
- "在您的团队里，做得好的人通常具备什么特质？"
  → 间接了解评估标准
  → 展示你想做好的态度

- "这个岗位在过去一年的发展中，有什么让您觉得特别有价值的项目？"
  → 展示你对工作内容有真实兴趣
```

**展示深度思考的问题：**

```
- "我注意到贵公司最近[某个动态/产品更新/战略调整]，
   这对我们团队的工作方向会有什么影响吗？"
  → 证明你做过功课

- "在[某个领域]，你们是偏向自研还是用开源/第三方方案？
   这个决策背后的考虑是什么？"
  → 展示你关心技术选型的trade-off思维

- "我的理解是这个岗位的核心价值在于[X]，这和您的期望一致吗？"
  → 展示你理解岗位本质
```

## 三、交叉面试官/跨部门面试

**适合问的：**

```
- "您的团队和[我应聘的团队]平时是怎么协作的？
   有没有协作上的挑战？"
  → 展示协作意识
  → 了解真实的跨部门关系

- "从您部门的角度，您最希望[我应聘的岗位]能提供什么支持？"
  → 展示你理解不同角色的需求

- "在您看来，公司文化中最特别的一点是什么？"
  → 跨部门面试官给的文化描述通常比HR更真实
```

---

## 四、终面/VP/高管

**适合问的（高格局问题）：**

```
战略方向：
- "您觉得我们这个业务在未来1-2年最大的机会在哪？"
- "行业正在经历[某变化]，您怎么看这对我们的影响？"
- "您做这个决策（创业/扩张/转型）最难的一点是什么？"

团队愿景：
- "您希望这个团队在接下来一年发展成什么样子？"
- "您选人最看重什么？"

个人层面（适度展示真诚）：
- "您在这家公司最有成就感的一个决策是什么？"
- "如果您能给一个即将加入的新人一条建议，会是什么？"
```

**不要问的：**
- ❌ 过于细节的执行层问题（终面官不关心这个）
- ❌ "您觉得我适合这个岗位吗？"（不要让终面官当场做判断）
- ❌ 薪资福利问题（除非终面官主动提起）

---

## 五、提问后的跟进技巧

```
面试官回答后，不要只说"好的，谢谢"。
可以简短回应，展示你在认真听：

示例：
  面试官："我们团队最大的挑战是数据质量不稳定。"
  你："理解，数据质量确实是很多数据驱动团队的底层痛点。
      我在上家公司也遇到过类似问题，当时我们的做法是[简短1-2句]。
      这个思路在您这边适用吗？"

这样做的效果：
  ① 证明你在认真听
  ② 自然展示你的经验
  ③ 把"提问"变成"对话"，印象分大增
```
