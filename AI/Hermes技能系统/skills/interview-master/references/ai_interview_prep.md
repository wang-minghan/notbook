# AI面试专项准备策略

---

## AI面试是什么？

AI面试通常由系统自动出题、录制你的回答（视频或音频），然后用算法评估。常见于校招初筛和大规模社招初筛。

**常见平台：** 牛客AI面试、HireVue、海纳AI、近屿智能等

---

## AI面试 vs 真人面试的核心差异

| 维度 | AI面试 | 真人面试 |
|------|--------|---------|
| 追问 | 不会追问，一次答完 | 会根据你的回答追问 |
| 反馈 | 没有面部反馈和互动 | 有眼神交流和反应 |
| 时间 | 每题严格限时（通常2-3分钟） | 相对灵活 |
| 评估 | 关键词+结构+语音特征 | 综合主观判断 |
| 重来 | 通常不能重录（部分允许重录1次） | 可以补充说明 |

---

## AI面试的评估维度

```
1. 内容质量（权重最高）
   - 关键词覆盖：你的回答是否命中了岗位相关的核心词
   - 结构完整性：是否有清晰的开头-中间-结尾
   - 案例具体度：有没有用STAR讲出具体案例

2. 表达能力
   - 语速稳定性：不要忽快忽慢
   - 流畅度：少用"嗯"、"那个"等填充词
   - 逻辑连接：句子之间有清晰的衔接

3. 非语言信号（视频面试）
   - 眼神：看镜头而不是看屏幕
   - 表情：自然、适度微笑
   - 姿态：坐姿端正，不要左右晃动
   - 背景：干净整洁，光线充足
```

## 应对策略

### 答题结构模板（万能版）

```
开头（10-15秒）：
  直接回答问题 + 预告你要讲几个点
  "关于这个问题，我的看法是[结论]。我从[A]和[B]两个方面来说明。"

中间（90-120秒）：
  用STAR讲1-2个案例
  每个案例控制在60秒以内
  关键词在前30秒内出现

收尾（10-15秒）：
  总结一句话，呼应开头
  "总结来说，[重申核心观点]。"
```

### 万能案例库准备

准备5个万能案例，覆盖以下维度：

```
维度1：团队协作 → 案例：[你的案例]
维度2：解决问题 → 案例：[你的案例]
维度3：主动性/领导力 → 案例：[你的案例]
维度4：抗压/时间管理 → 案例：[你的案例]
维度5：学习/成长 → 案例：[你的案例]

每个案例提前写好STAR，练到能在2分钟内自然讲完。
```

### 技术准备清单

```
硬件：
  □ 摄像头清晰、麦克风收音正常
  □ 网络稳定（建议有线连接）
  □ 关掉所有可能弹通知的应用
  □ 手机静音

环境：
  □ 安静的房间，不会被打扰
  □ 背景简洁干净（白墙或书架）
  □ 正面光源充足（不要逆光）
  □ 摄像头与眼睛齐平

个人：
  □ 穿正式或商务休闲装（至少上半身）
  □ 提前测试设备和平台
  □ 准备纸笔（可以快速记录题目要点）
```

### 练习方法

```
1. 手机录像法
   用手机录自己回答常见问题，回放检查：
   - 有没有"嗯"、"那个"等口头禅
   - 眼神是否在看镜头
   - 回答结构是否清晰
   - 时间是否在2-3分钟内

2. 模拟面试
   找朋友用AI面试的方式提问你：
   - 不给追问，让你一次答完
   - 严格计时
   - 结束后给反馈

3. 关键词卡片法
   每个岗位准备10-15个核心关键词卡片
   练习时确保这些词自然出现在回答中
```

## 常见AI面试题型

### 1. 自我介绍（几乎必考）
控制在90秒，重点突出和岗位相关的经历。

### 2. 行为问题（高频）
和传统行为面试一样，但你没有追问机会，所以STAR每一步都要讲完整。

### 3. 情景题
"如果遇到[某情况]，你会怎么做？"
按"分析 → 行动 → 预期结果"的顺序回答。

### 4. 专业知识题（部分平台）
技术岗可能有编程题或专业知识问答。

### 5. 开放题
"你怎么看待[某趋势/某现象]？"
用"观点 → 2-3个论据 → 总结"的结构。

## 关键提醒

- **不要背答案：** AI能检测到机械背诵（语速过于均匀、缺少自然停顿）
- **看镜头，不看屏幕：** 视频面试中，看镜头=看面试官眼睛
- **不要过度紧张：** AI不会因为你紧张一下就扣分，保持整体流畅即可
- **注意时间：** 到时间会被直接截断，一定要在时限内完成核心内容
- **如果可以重录：** 只在严重失误时重录，微小瑕疵不用重来
