## Obsidian Journals （翻译）

适用于 [Obsidian](https://obsidian.md/) 的全面日记解决方案，可改变您的笔记记录体验。此插件可帮助您跨多个时间范围（从每日笔记到自定义期间）创建、组织和导航结构化日记账分录。无论您是跟踪日常工作日志、按周组织研究笔记，还是管理项目冲刺，Journals 都提供了强大的工具，可实现基于时间的笔记的一致格式、轻松导航和可视化组织。

## 主要特点

### 日记帐类型

-   **标准间隔**：每日、每周、每月、季度和每年笔记
-   **自定义期间**：为任何自定义持续时间（冲刺、财务季度）创建注释
-   **索引支持**：自动编号条目（Sprint 1、Sprint 2、Quarter 3）
-   **多个日记**：为您生活的不同方面设置不同的日记

### 视觉和导航

-   **日历视图**：内置日历，便于日期导航
-   **Timeline View**：使用可自定义的代码块在时间段中导航
-   **音符装饰**：根据内容、日期或状态直观地突出显示音符
-   **导航块**：指向相关日记账分录的快速链接

### 组织

-   **日记帐功能区**：将相关日记帐分组在一起
-   **自定义命令**：为最常见的日志作创建快捷方式
-   **模板：**强大的变量系统，实现一致的日记账分录
-   **Frontmatter**：用于更好组织的自动元数据

## 安装

请按照以下步骤安装插件。

1.  在 Obsidian 的社区插件浏览器中搜索 “Journals”
2.  在您的 Obsidian 设置中启用该插件（在 “Community plugins” 下找到 “Journals”）。
3.  检查设置。配置您需要的日记帐。

## 设置

Journals 插件提供了广泛的配置选项来自定义您的日记体验。本节介绍您将遇到的主要设置。

### 全局设置

-   **使用功能区**：启用此选项可将日记帐组织到逻辑组（如工作、个人、项目）中。搁架允许您：
    
    -   筛选日历视图以仅显示特定功能架中的日志
    -   将导航块中的装饰限制在当前搁架上
    -   一起管理相关日记账
-   **Open on Startup （启动时打开）：**选择启动 Obsidian 时将自动打开的日志。选择“不打开”以禁用此功能。
    

### 日历设置

-   **周开始**时间：选择一周的开始日期（星期一、星期日等）
-   **日历视图位置**：在左侧或右侧边栏中显示日历
-   **周数**：显示工作日之前或之后的周数，或隐藏它们
-   **今日按钮模式**：
    -   创建注释：如果今天的注释不存在，则创建该注释
    -   导航：仅打开现有笔记
    -   切换日期：仅将日历视图切换到当前月份
-   **日历突出显示**：
    -   自定义今天的日期突出显示（文本和背景颜色）
    -   自定义活动注释高亮显示（文本和背景颜色）

### 日志配置

可以使用以下设置单独配置每个日志：

-   **注释创建**：
    -   名称模板：设置音符文件名模式
    -   日期格式：定义日期的显示方式
    -   文件夹路径：笔记的存储位置
    -   模板：为注释内容选择模板
    -   自动创建：自动创建当前期间的注释
    -   确认创建：创建备注前提示
-   **年表设置**：
    -   Start date：此日记帐的开始时间
    -   结束条件：何时停止创建注释（从不、特定日期、N 次重复后）
-   **索引设置**：对于编号条目（如 “Sprint 1”）
    -   启用索引
    -   设置锚定日期和初始索引
    -   选择增量类型（连续或在 N 个条目后重置）
-   **Frontmatter**：自定义 frontmatter 字段
    -   日期字段名称
    -   添加开始/结束日期
    -   索引字段名称

### 装饰系统

Journals 提供了一个强大的装饰系统，可以在日历视图和导航块中直观地区分笔记：
-   **装饰**：每个日记可以有多个装饰，根据条件直观地标记笔记
-   **条件**：根据以下因素应用装饰：
    -   **注释内容**：标题包含文本、具有特定标签、frontmatter 属性
    -   **基于日期**：特定日期、工作日、相对偏移量
    -   **备注状态**：有备注、有未结任务、所有任务已完成
-   **样式**：自定义外观：
    -   背景颜色
    -   文本颜色
    -   边框（统一或具有自定义宽度、颜色、样式的每边）
    -   形状（各种位置的正方形、圆形、三角形）
    -   角标记
    -   图标（来自 Obsidian 的图标集）
-   **组合**：使用 AND 逻辑（所有条件必须匹配）或 OR 逻辑（任何条件都可以匹配）

### 自定义命令

创建自定义命令以在日记帐注释之间快速导航：

-   **命令类型**：
    -   当前/下一个/上一期间的同一日期（例如，“下周同一天”）
    -   同一日记账中的下一个/上一个条目
    -   组合导航（例如，“下个月的同一天”）
-   **上下文设置**：
    -   今天：命令始终可用，以今天作为参考
    -   打开笔记：打开任何笔记时可用的命令
    -   Only open note（仅打开注释）：Command 仅在此日志中的注释打开时可用
-   **UI 集成**：
    -   使用自定义图标添加到功能区栏
    -   打开模式（活动窗格、新选项卡、拆分、窗口）

### 导航块

配置有助于在日记账条目之间导航的代码块：

-   **区块类型**：
    -   导航块：相对于当前注释导航
    -   日历视图块：显示当前期间的日历
-   **入口行定制**：
    -   包含变量的模板文本（如日期格式）
    -   字体样式（大小、粗体、斜体）
    -   颜色（文本和背景）
    -   链接行为（链接到自身、当前日记帐或特定期间）
    -   将视觉装饰应用于条目
-   **设置**：
    -   创建模式（创建缺失的注释或仅显示现有注释）
    -   整块装饰

## 与其他插件的兼容性

-   `Daily notes`核心插件 - 此插件旨在替代它。通过 Daily notes 创建的笔记不会连接到任何日志，因此建议禁用此插件。
-   `Periodic Notes`community plugin - 这个插件最初的灵感来自似乎被遗弃的 Periodic notes，旨在成为它的替代品。
-   `Calendar`社区插件 - 从 1.1.0 开始，此插件具有类似于日历插件的日历视图。截至目前，它还不具备日历插件的所有功能，但会逐渐迎头赶上。
-   `Templater`社区插件 - 从 1.3.0 开始，插件在其设置中支持 Templater 模板。日志插件变量首先被替换，可以在 templater 命令中使用。

### Templater caveats

在某些情况下，Templater 开始干扰插件作，导致从 frontmatter 中删除部分损坏的笔记或日志相关数据。 避免此类问题的最佳设置是：

-   在 Journal Plugin Settings 中配置的模板
-   `Trigger Templater on new file creation`已禁用
-   OR 已启用、已启用、**未**配置文件夹模板`Trigger Templater on new file creation``Enable Folder Templates`

这确保了只有日志插件在处理注释模板，从而避免与 templater 插件冲突（日志插件将在后台使用 templater 本身来处理 templater 命令）。

## 高级用法

### 使用工具架

书架是将日记帐组织成逻辑组的强大方法。以下是有效使用它们的方法：

1.  在全局设置中**启用 Shelves**。
2.  为您生活的不同领域**创建 Shelves**：
    -   工作（会议、项目、周报）
    -   个人（日记、习惯跟踪、健康）
    -   教育（课程、研究、学习笔记）
3.  在创建或编辑日志时将 **Journals （日记） 分配给 Shelf （将日记账分配给书架**）。
4.  按功能区**筛选 Calendar View** 以专注于特定上下文。
5.  使用 journals-home 代码块中的 **Shelf Parameter** 为特定上下文创建仪表板注释。

### 常见设置示例

#### 每日工作日志
```yaml
Type: Day
Folder: Work/DailyNotes
Name Template: {{date}} Daily Log
Date Format: YYYY-MM-DD
Auto-create: Enabled
Start date: Your employment start date
```

#### 项目冲刺

```yaml
Type: Custom
Every: Week
Duration: 2
Folder: Projects/{{journal_name}}/Sprints
Name Template: Sprint {{index}}
Index: Enabled, Increment type
```

#### 学术学期笔记

```yaml
Type: Week
Folder: Education/{{date:YYYY}}/{{date:MMMM}}
Name Template: Week {{index}} - {{start_date:MMM D}} to {{end_date:MMM D}}
Start date: Term start date
End conditions: Date (term end date)
```

## 故障 排除

### 常见问题

#### 在错误的位置创建了注释

-   在日志设置中检查文件夹路径
-   验证您的注释名称模板不包含非法字符
-   确保该文件夹存在于您的 Vault 中

#### 模板变量不起作用

-   验证语法：使用双括号`{{variable}}`
-   对于日期格式，请使用 Moment.js 语法（例如`{{date:YYYY-MM-DD}}`)
-   检查变量名称中的空格或拼写错误

#### 日历视图不显示备注

-   确保注释具有正确的 frontmatter （日志名称和日期）
-   检查您是否按书架筛选，以及日志是否已分配给该书架
-   验证日记帐设置中的日期格式是否与备注日期匹配

#### 与 Templater 冲突

-   按照兼容性部分中的 Templater 设置
-   确保 Templater 未配置为自动处理相同的模板
-   推荐的设置是让 Journal 插件处理模板

#### 缺少装饰品

-   验证您的条件条件 （标记、属性、日期）
-   检查 OR 是否在更合适时使用 AND 逻辑
-   确保装饰样式设置已正确配置
