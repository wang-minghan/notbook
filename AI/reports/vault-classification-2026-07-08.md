# Vault Classification Report 2026-07-08

## 主知识库粗分层
- `entity`：工作经历、教育经历、证书、读书笔记、面试叙事页
- `concept`：RAG、混合检索、BM25、RRF、Cross Encoder、Neo4j、vLLM、知识库生命周期、面试叙事主线
- `query`：面试提问整理、计划类页面
- `index`：AI 总览、总索引、教育总览
- `log`：AI 变更日志
- `schema`：AI 结构规范

## 这次最重要的判断
1. 你的库已经不是“资料堆”，而是有了主干与枢纽。
2. 真正需要继续补的是：
   - query / comparison 的回写
   - 半孤岛页的自然链接
   - 计划页的结构化升级
3. `AI/Hermes技能系统` 这类子库不应混入主知识库分类，它们更像独立项目区。

## 下一步建议
- 把 `工作/面试准备/面试问答整理.md` 和 `面试叙事主线.md` 继续互相双向链接
- 把读书与计划页逐步转成 comparison / query / concept
- 继续让自动整理任务优先处理半孤岛，而不是只报孤岛
- 为关键概念页补更多双向链接
