# 亚信科技（中国）有限公司 — 人工智能算法工程师

**时间：** 2025.08 — 2026.03  
**职位：** 人工智能算法工程师  
**地点：** 成都

---

## 核心职责

主导设计并落地 **基于多Agent架构的大模型推理增强系统**，针对复杂业务场景中出现的语义漂移、多跳推理不稳定和幻觉问题，构建本体驱动的知识建模与约束推理体系。

## 技术架构

### Agent 架构设计（LangGraph 思想）
- 任务流拆解为多个协作 Agent：
  - 查询解析 Agent
  - 混合检索 Agent
  - 本体对齐 Agent
  - 受约束推理 Agent
  - 一致性校验 Agent

### 检索增强（Hybrid Retrieval）
- **BM25**（Elasticsearch）关键词检索
- **BGE 向量检索** 语义匹配
- **RRF 融合** + **Cross Encoder 重排** 提升召回质量

### 知识建模（Neo4j）
- 构建业务本体与关系结构
- 推理阶段引入**关系合法性校验**与**路径裁剪机制**
- 降低模型幻觉率，增强结果可解释性

### 模型侧
- **Qwen 系列模型** + **LoRA** 领域微调
- **vLLM** + 混合精度优化推理
- **Docker + Kubernetes** 容器化部署

### 语义指令映射
- 将业务规则与操作逻辑映射为可组合的语义指令单元
- 实现自然语言到结构化指令的映射与执行

## 主要业绩

1. 构建基于**本体约束的推理链建模方法**，实现大模型推理路径的显式表达与可视化分析，支持逐层验证
2. 通过语义约束、关系一致性校验与上下文收敛机制，**系统性抑制模型幻觉**，在多轮推理与复杂业务问答中显著提升稳定性与一致性
3. 在对比传统 RAG 与纯向量检索的实验中获得**明显优势**（准确率、推理一致性、语义泛化能力）
4. 成功推进客户交付，具备实际产品落地经验

## 技术栈

`PyTorch` `LangChain/LangGraph` `Qwen` `LoRA` `vLLM` `Neo4j` `Elasticsearch` `BGE` `Docker` `Kubernetes`
