## 1.1.2 Update

- 主页新增每日金句
	- 这个功能可以随机显示打了特定标签的段落文字。目前主页里设置的参数是 `#content/观点` 标签，只要笔记里有段落打了这个标签，就有机会出现在主页开头的位置。预览模式下的刷新频率取决于dataview设置里的刷新频率，阅读模式下关闭再打开也会刷新。
	- 使用方法见：[[【脚本说明】random-quote]]
- 主页添加全年时间线装饰条
	- 时间线上的小人会根据日期自动变化位置，也可以通过参数在特定日期位置添加喜欢的图标。
	- 添加图标的语法范例：`await dv.view("year-timeline-1", { events: [['5/5', '日期说明1', '♉'], ['6/17', '日期说明2', '♊'], ['12/4', '日期说明3', '♐']]})`
- 任务中心新增任务热力图
	- 使用方法见：[[【脚本说明】任务热力图]]
- 其他一些小更新
	- 任务中心完善了2个比较常用的任务查询场景，进行中任务（Active Tasks）和已过期任务（Overdue Tasks），未来会再添加GTD中心（基于场景的任务查询）。
		- 进行中任务会包含：已经开始且尚未到截止日期的任务
		- 已过期任务会包含：在当前日期已经截止但仍未完成的任务

- 任务中心更新
	- 任务中心新增任务统计
		- Vault Todo（库待办任务）：统计所有没有标记符号的待办
		- Postponed（推迟任务）：标记为 `- [>]` 的待办任务
		- Information（非任务提醒）：标记为 `- [i]` （信息） 或 `- [n]` （备注） 或 `- [!]` （提醒） 的任务

		- Vault Done （已完成任务）
	- 新增按文件夹任务查询标签视图

关于任务中心统计的说明：
- 任务中心的数据统计时基于dataviewjs行内语句查询，而标签视图中查询的任务则是基于task插的查询语法。尽管是按文件夹进行的分类，理论上数据统计跟查询到的任务总和应该数据量一致，但是Tasks插件确实存在部分任务查询不到的情况，具体原因不清楚。
- 如所有任务都清空，但VaultTodo仍显示有未完成任务，可以使用 900 Assets/920 Queries 里的 `TaskCenterBlock` 笔记查询验证未被查询到的任务是什么。
- `TaskCenterBlock` 中的查阅是基于dataviewjs编写的，准确度高，但语法复杂，修改比较麻烦，日常仍然推荐使用tasks查询的语法。
