# Agent编排

Agent 编排指的是把复杂任务拆成多个职责清晰的 Agent 协作执行，而不是让一个 LLM 从头干到尾。

## 你项目里的结构
- 查询解析 Agent
- 混合检索 Agent
- 本体对齐 Agent
- 受约束推理 Agent
- 一致性校验 Agent

## 为什么这么做
- 分工明确
- 复杂流程更稳定
- 便于调试和替换子模块
- 更适合多跳推理和带约束的任务
